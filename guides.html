🍪 Cookie-Based Daily Claiming

To let the bot claim daily bonuses for you, it needs your login cookies. This is a one-time setup per casino. Here's how to grab them from your browser:

Getting Your Cookies

1

Log in to the casino in your browser (Chrome, Firefox, Edge).

2

Open Developer Tools (F12) → Application tab → Cookies → select the casino domain.

3

Copy the cookie string. Right-click any cookie → "Copy All" or paste this in the address bar and press Enter:

javascript:prompt('Cookies:', document.cookie);
4

Save to the bot using the /setcookie command:

🎬 Streamer Alerts

Never miss a link drop during a live stream. The bot watches Kick.com streamer chats and forwards any links to your streamer channel.

How to Set Up

Edit data/streamer_profiles.json and add the streamers you want to follow:

[
  {
    "name": "My Streamer",
    "enabled": true,
    "kickUsername": "streamername",
    "monitorLinks": true
  }
]

Set enabled: true and restart the bot — it will connect to that streamer's chat on startup.

ⓘ

Streamer alerts use their own channel and dedup system, separate from Reddit monitor posts.

🤖 Auto-Claim (Scheduled)

Schedule

️‍🔧 Troubleshooting

Check the bot's console window. Common causes:

  • Channel IDs in the settings file are wrong — check with /status
  • The RSS filter isn't matching any current posts
  • The bot doesn't have permission to send messages in the target channel

Make sure "enabled": true in the streamer profile and the Kick username is correct. Check that STREAMER_CHANNEL_ID points to a channel the bot can see. The console will show connection status.

Restart the bot. If it continues, check the console for the actual error — it will show a stack trace. Common causes: missing settings fields or network timeouts.

Commands register instantly on your server. If they don't appear, re-invite the bot with the applications.commands OAuth2 scope. Type / and wait a moment for Discord to sync.

Check the console for clues. Common causes:

  • Discord API rate limiting — the bot reconnects automatically
  • Kick WebSocket disconnect — auto-reconnects after 10 seconds
  • Invalid JSON in config files — validate with a JSON checker
  • RAM usage — the bot uses ~100 MB under normal load
